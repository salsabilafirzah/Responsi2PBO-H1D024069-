Pertanyaan Refleksi
Setelah menyelesaikan kode, jawab pertanyaan berikut:

1. Apa fungsi keyword 'extends' dalam deklarasi class Java?
2. Jelaskan perbedaan antara Method Overloading (yang ada di latihan sebelumnya) dengan Method Overriding (yang ada di latihan ini)!
3. Apa kegunaan keyword 'super' di dalam constructor subclass (Manajer)?
4. Jika variabel di class Karyawan diubah access modifier-nya menjadi 'private', apa yang akan terjadi pada class Manajer? Bagaimana solusinya?
5. Mengapa kita perlu menggunakan Inheritance? Apa keuntungan utamanya dibandingkan membuat dua class yang terpisah sama sekali (misal class Karyawan dan class Manajer dibuat terpisah tanpa hubungan)?

Jawaban:

1. extends dipakai untuk menunjukkan bahwa sebuah class mewarisi class lain. Jadi class yang ditulis
   setelah extends adalah parent (superclass), dan class yang menuliskannya adalah child (subclass).
   Dengan extends, subclass bisa memakai atribut dan method yang sudah dimiliki superclass tanpa harus
   menulis ulang.

2. Method Overloading: terjadi dalam satu class yang sama. Nama methodnya sama, tapi parameternya
   berbeda (jumlah/tipe/urutan). Tujuannya untuk memberikan pilihan pemanggilan method dengan input
   berbeda.
   Method Overriding: terjadi pada dua class yang punya hubungan inheritance. Subclass menulis ulang
   method yang sudah ada di superclass dengan tujuan menyesuaikan perilakunya. Nama method, parameter,
   dan tipe return harus sama.

3. super(...) digunakan untuk memanggil constructor milik superclass lebih dulu. Ini penting supaya
   bagian data yang dimiliki parent bisa diinisialisasi dengan benar sebelum subclass menambahkan bagian
   miliknya sendiri. Tanpa super, data parent bisa jadi kosong atau tidak di-set.

4. Kalau variabel di Karyawan dibuat private, berarti class Manajer tidak bisa mengakses langsung
   variabel tersebut, karena private hanya boleh diakses dari dalam class itu sendiri.
   Solusinya: sediakan getter dan setter pada class Karyawan. Jadi Manajer tetap bisa mengambil atau
   mengubah data, tapi lewat method yang terkontrol (sesuai prinsip encapsulation).

5. Dengan inheritance, kita bisa menghindari duplikasi kode. Fitur umum yang sama cukup dibuat
   sekali di superclass, lalu subclass tinggal pakai. Kalau suatu saat ada perubahan logika, cukup
   ubah di satu tempat saja.
